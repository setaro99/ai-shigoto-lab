# AI仕事術ラボ（ブログ）

事務・ビジネスの「今日の仕事」をAIで短くする実践ハウツーを配信するJekyllブログ。GitHub Pages で公開しています。

**公開URL**: https://setaro99.github.io/ai-shigoto-lab/

---

## 編集方針（記事を書くときの規範）

- 1記事2000〜3500字。**検索者の悩みを実際に解決する実用ハウツー**だけを書く
- **コピペでそのまま使えるプロンプト**を必ず1つ以上含める（半角バッククォート3つのコードフェンスで囲む）
- 構成: 読者の悩み → 解決の考え方 → コピペ用プロンプト → 使う前/送る前のチェック → まとめ
- 誇大な効果保証は書かない。医療・法律・投資・税務の専門的助言はしない
- 実在ツールの仕様はバージョン非依存の書き方にする（ChatGPT/Copilot/Gemini/Claudeのどれでも動く形）
- **記事末に必ず「本への導線」を付ける**（下の専用セクション参照）。テキストだけの言及で終わらせず、**必ずAmazonリンクを貼る**
- 広告・アフィリエイトリンクを含む記事は、**冒頭に「PR」を明記**する（景品表示法・ステマ規制の順守）。※自著Kindleへのリンクは第三者広告ではないためPR表記は不要（著者名を明記しているので出所は自明）
- 既存記事とテーマ・見出しが重複しないよう必ず確認する

---

## 本への導線の書き方（記事末に必須・収益の要）

記事の `---` 区切りのあと、**2段落**で書く。1段落目＝記事テーマに最も近い1冊をリンク付きで、2段落目＝他の巻＋本の一覧ページへ。

**テーマ→本の対応**（記事テーマに一番近い巻を1段落目に置く）:

| 記事テーマ | 紹介する本 | Amazonリンク |
|---|---|---|
| メール全般・議事録・報告・部下対応・会議 | 『AI議事録・メール術』 | `https://www.amazon.co.jp/dp/B0H92BMX67` |
| Excel・関数・VBA・パワークエリ・データ整形 | 『ExcelをAIに書かせる本』 | `https://www.amazon.co.jp/dp/B0H93XVJP1` |
| 事務全般・資料作成・マニュアル・引き継ぎ | 『残業ゼロのAI仕事術』 | `https://www.amazon.co.jp/dp/B0H7MLFPFM` |
| 営業・商談・提案書・顧客フォロー | 『営業のためのAI仕事術』 | ※審査中。発売までは上記から近いものを選ぶ |

**書き方の例**（Excelの記事の場合）:

```markdown
---

この記事のような「日本語で頼むだけ」のExcelレシピを、28本まとめた本があります。複数条件の集計、#N/Aの原因特定、VBAマクロ、パワークエリまで——**[『ExcelをAIに書かせる本』](https://www.amazon.co.jp/dp/B0H93XVJP1)**（著: イイヅカ ミナト／¥499・Kindle Unlimited会員は¥0）。

事務職向けの『残業ゼロのAI仕事術』、管理職向けの『AI議事録・メール術』もあります → **[本の一覧]({{ '/books/' | relative_url }})**
```

ルール:
- 1段落目は「この記事の続き・体系版」として自然に繋ぐ。誇大表現や煽りは書かない
- **「¥499・Kindle Unlimited会員は¥0」を必ず添える**（KU読み放題はKENP収益になるため訴求価値が高い）
- 2段落目の本の一覧リンクは必ず `{{ '/books/' | relative_url }}` を使う（baseurl対応のため）
- 新刊が出たら `books.md` と上の表を更新する

## 記事フォーマット

`_posts/YYYY-MM-DD-slug.md`（slugは半角英数ハイフン）

```
---
layout: post
title: "検索キーワードを含む32字以内のタイトル"
date: YYYY-MM-DD 06:00:00 +0900
categories: [メール]
tags: [お詫びメール, ChatGPT, プロンプト]
description: "検索意図に答える120字前後の説明文"
---
```

## テーマ計画（上から順に執筆・書き終えたら新テーマを追加）

事務職・ビジネスパーソンの検索需要が見込める、具体的な悩み起点のロングテールKW。

1. ~~AIでお詫びメールを書く｜コピペで使える例文とプロンプト~~（公開済み）
2. ~~議事録をAIで清書する方法｜走り書きメモから10分で~~（公開済み）
3. ~~Excelの関数をAIに作らせる｜「日本語で頼むだけ」入門~~（公開済み）
4. ~~角の立たない断りメールの書き方｜AIプロンプト付き~~（公開済み）
5. ~~AIで作る業務マニュアル｜質問に答えるだけで完成~~（公開済み）
6. ~~企画書のたたき台をAIで作る｜構成から一気に~~（公開済み）
7. ~~英語メールをAIで読む・返す｜翻訳より速い方法~~（公開済み）
8. ~~会議のアジェンダをAIで作る｜時間配分つきテンプレ~~（公開済み）
9. ~~VBAマクロをAIに書かせる｜未経験でも業務自動化~~（公開済み）
10. 引き継ぎ書をAIで10分｜抜け漏れを防ぐコツ

すべて書き終えたら、既存記事と重複しない新しいロングテールKW（職種特化・場面特化）を立てて継続する。

## ローカルでプレビューする場合

```bash
bundle install
bundle exec jekyll serve
```

## サイト構成

- `_config.yml` — サイト設定
- `index.md` — トップページ
- `about.md` — 運営者情報
- `books.md` — **Kindleシリーズ一覧（収益導線のハブ。新刊が出たら必ず更新）**
- `_posts/` — 記事
