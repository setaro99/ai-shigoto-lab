# AI仕事術ラボ（ブログ）

事務・ビジネスの「今日の仕事」をAIで短くする実践ハウツーを配信するJekyllブログ。GitHub Pages で公開しています。

**公開URL**: https://setaro99.github.io/ai-shigoto-lab/

---

## 編集方針（記事を書くときの規範）

- 1記事2000〜3500字。**検索者の悩みを実際に解決する実用ハウツー**だけを書く
- **コピペでそのまま使えるプロンプト**を必ず1つ以上含める（半角バッククォート3つのコードフェンスで囲む）
- 構成: 読者の悩み → 解決の考え方 → コピペ用プロンプト → 使う前/送る前のチェック → まとめ
- 誇大な効果保証は書かない。医療・法律・投資・税務の専門的助言はしない
- 実在ツールの仕様はバージョン非依存の書き方にする（ChatGPT/Copilot/Gemini/Claudeのどれでも動く形）
- 記事末に、Kindleシリーズ「AI仕事術ラボ」（著: イイヅカ ミナト）への自然な案内を1段落
- 広告・アフィリエイトリンクを含む記事は、**冒頭に「PR」を明記**する（景品表示法・ステマ規制の順守）
- 既存記事とテーマ・見出しが重複しないよう必ず確認する

## 記事フォーマット

`_posts/YYYY-MM-DD-slug.md`（slugは半角英数ハイフン）

```
---
layout: post
title: "検索キーワードを含む32字以内のタイトル"
date: YYYY-MM-DD 06:00:00 +0900
categories: [メール]
tags: [お詫びメール, ChatGPT, プロンプト]
description: "検索意図に答える120字前後の説明文"
---
```

## テーマ計画（上から順に執筆・書き終えたら新テーマを追加）

事務職・ビジネスパーソンの検索需要が見込める、具体的な悩み起点のロングテールKW。

1. ~~AIでお詫びメールを書く｜コピペで使える例文とプロンプト~~（公開済み）
2. 議事録をAIで清書する方法｜走り書きメモから10分で
3. Excelの関数をAIに作らせる｜「日本語で頼むだけ」入門
4. 角の立たない断りメールの書き方｜AIプロンプト付き
5. AIで作る業務マニュアル｜質問に答えるだけで完成
6. 企画書のたたき台をAIで作る｜構成から一気に
7. 英語メールをAIで読む・返す｜翻訳より速い方法
8. 会議のアジェンダをAIで作る｜時間配分つきテンプレ
9. VBAマクロをAIに書かせる｜未経験でも業務自動化
10. 引き継ぎ書をAIで10分｜抜け漏れを防ぐコツ

すべて書き終えたら、既存記事と重複しない新しいロングテールKW（職種特化・場面特化）を立てて継続する。

## ローカルでプレビューする場合

```bash
bundle install
bundle exec jekyll serve
```

## サイト構成

- `_config.yml` — サイト設定
- `index.md` — トップページ
- `about.md` — 運営者情報
- `_posts/` — 記事
